# Navigation Reorganization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reorganize the authenticated-user navigation (top nav, dropdown, mobile slide-out) into a cleaner, grouped structure; make Court Compliance always visible to drive upsell discoverability; consolidate Medallions and Community/Groups via tab navigation.

**Architecture:** Most changes live in `templates/base.html` (top nav + dropdown + mobile slide-out) and `static/css/base-inline.css` (new pill class + section label styles). Two small reusable template partials (`_medallions_tab_nav.html`, `_community_tab_nav.html`) get included at the top of the related pages so users can switch between consolidated views. No URL changes, no view changes, no migrations — keep existing URLs to preserve SEO and bookmarks.

**Tech Stack:** Django templates, vanilla CSS, no JS or new dependencies.

**Reference spec:** `docs/plans/2026-05-26-nav-reorg-design.md`
