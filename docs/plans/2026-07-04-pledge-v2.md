# Pledge v2 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the daily pledge based on founder dogfooding: (1) per-user timezone so "today" is the user's real day (bug: a pledge made in the evening was stamped the next UTC day), (2) sync the two pledge surfaces (progress home card + check-in page) into one pledge, (3) let a completed pledge carry an optional note + photo, editable after the fact, (4) share a pledge to the in-app feed and externally to social media with a myrecoverypal.com link.

**Architecture:** Add `User.timezone` (IANA string) captured from the browser and activated per request by a middleware, then switch all pledge/check-in "today" computations from `timezone.now().date()` to `timezone.localdate()`. Add `note` + `photo` to `DailyPledge`. Both pledge surfaces read `pledged_today`/`pledge_streak` from the same source and POST to the same endpoints. Sharing reuses the existing `create_social_post` image pipeline (feed) and the existing `[data-share-url]` → `MRPNative.share`/`navigator.share` interceptor (external).

**Tech Stack:** Django 5.0.10, PostgreSQL, Cloudinary (media), `zoneinfo` (stdlib, tz validation), vanilla JS + fetch, Capacitor share plugin. Use `python3`; run tests locally (no `SENTRY_DSN`); never `railway run`.

**Parent:** Follows the shipped daily-pledge feature (`docs/plans/2026-07-03-daily-pledge.md`). Roadmap Phase 2 (retention). Decisions locked with the founder 2026-07-04: sync the two surfaces; per-user timezone; add note + photo + share-to-feed + external social share (link to myrecoverypal.com).

## Global Constraints

- Match existing repo style: function-based views, plain `request.POST`/`request.FILES`, feature tests in `apps/accounts/tests_pledge.py`. Client tests hitting the test client need `@override_settings(PREPEND_WWW=False, SECURE_SSL_REDIRECT=False)`.
- **Never gate the pledge, note, photo, or share behind Premium.** All free.
- The pledge stays in `DailyPledge` (one row per user per **user-local** day). One-tap pledge must never create/mutate a `DailyCheckIn`.
- "Today" everywhere in pledge/check-in logic = `timezone.localdate()` (which reflects the per-request activated user timezone). Never `timezone.now().date()` for pledge/check-in day math after this plan.
- Timezone strings must be validated against `zoneinfo.available_timezones()` before storing; reject unknown values (never `timezone.activate` an unvalidated string).
- Photo uploads reuse `apps/accounts/image_utils.py` (`validate_image`, `process_uploaded_image`) exactly as `create_social_post` does — compress only when `not is_cloudinary_enabled()`.
- External share URL is always `https://www.myrecoverypal.com` (growth: backlink + brand). Share text must not leak private data beyond the pledge sentence, day count, and optional user-authored note.
- Mobile: all new UI works in the Capacitor WebView.

**Key design decisions locked in:**
1. **Per-user timezone is the source of truth for "today."** Middleware activates `user.timezone`; `timezone.localdate()` then returns the user's local date for both server-rendered state (`pledged_today`, streak) and writes. The check-in's existing client `local_date` input stays as a first-request fallback (before tz is stored) but is no longer the primary mechanism.
2. **`DailyPledge.note`/`photo` are per-day**, distinct from the enduring `User.pledge_reason`/`pledge_photo` (card identity). Both render on the card.
3. **One pledge/day, two windows.** Progress card and check-in card both read `pledged_today`/`pledge_streak` and POST to `pledge_today`/`update_pledge`/`share_pledge_to_feed`.
4. **Share-to-feed reuses `create_social_post`'s image pipeline**; external share reuses the `[data-share-url]` native/`navigator.share` interceptor already in `capacitor-native.js`.

```python
from django.utils import timezone as dj_timezone

class UserTimezoneMiddleware:
    """Activate the authenticated user's stored IANA timezone so
    timezone.localdate() reflects their real local day (streaks/pledges)."""
    def __init__(self, get_response):
        self.get_response = get_response

    def __call__(self, request):
        tz = getattr(getattr(request, 'user', None), 'timezone', '') or ''
        activated = False
        if tz:
            try:
                dj_timezone.activate(tz)
                activated = True
            except Exception:
                dj_timezone.deactivate()
        try:
            return self.get_response(request)
        finally:
            if activated:
                dj_timezone.deactivate()
```

```python
@login_required
@require_POST
def set_timezone(request):
    import json, zoneinfo
    try:
        tz = (json.loads(request.body or '{}').get('timezone') or '').strip()
    except (ValueError, TypeError):
        tz = ''
    if tz and tz in zoneinfo.available_timezones():
        if request.user.timezone != tz:
            request.user.timezone = tz
            request.user.save(update_fields=['timezone'])
        return JsonResponse({'success': True})
    return JsonResponse({'success': False}, status=200)
```
